# FreeProcurement
Данная система является открытым ПО для управления обеспечением удовлетворения внутренних потребностей организации путем управления закупками.
# ИС «Закупки-Бюджет»
Информационная система управления закупками и контроля исполнения бюджета. Система предназначена для автоматизации полного цикла закупочной деятельности в организации. Область применения охватывает все процессы: от формирования внутренней потребности подразделений до этапа размещения лота на закупку.
## Основной функционал
*   Централизованная регистрация, хранение и маршрутизация заявок на закупку от подразделений.
*   Автоматизированная проверка заявок на соответствие утверждённым лимитам бюджетной статьи подразделения на текущий период.
*   Формирование лотов для проведения закупки на основе утверждённых заявок.
*   Выполнение план-фактного анализа исполнения бюджета и визуализация данных с помощью динамических графиков и диаграмм.
*   Отправка уведомлений инициаторам при изменении статусов заявок.
## Роли пользователей
Система поддерживает разделение прав доступа для следующих групп пользователей:
*   **Инициатор (штатный работник):** формирование первичной потребности и отслеживание статуса заявок.
*   **Руководитель подразделения:** согласование, валидация и утверждение заявок.
*   **Менеджер по закупкам:** формирование лотов из заявок и их размещение на закупку.
*   **Администратор:** управление пользователями, справочными данными и правами доступа.
## Технологический стек
Проект реализован с использованием трёхуровневой архитектуры.
**Frontend (Уровень представлений):**
*   Фреймворк Nuxt.js с использованием серверного рендеринга (SSR).
*   HTML, CSS, JavaScript.
**Backend (Уровень приложений):**
*   Язык программирования: Python 3.10.
*   Фреймворк: Django 4.2.
*   REST API: Django REST Framework.
*   Генерация документов: библиотека `docxtpl` (поддержка синтаксиса Jinja2).
*   Фоновые задачи: Celery / Redis (для тяжелых отчетов и массовой рассылки).
**База данных (Уровень данных):**
*   СУБД: PostgreSQL 14.
*   Драйвер: `psycopg2`.
*   Локальное файловое хранилище для работы с шаблонами и документами.
## Архитектура и интеграция
*   Обмен данными между клиентской и серверной частью осуществляется через REST API по протоколу HTTP/HTTPS в формате JSON.
*   Серверная часть обрабатывает бизнес-логику (паттерн Data Mapper) и обращается к базе данных через встроенный механизм ORM.
*   Реализована возможность интеграции с внешними информационными системами (Битрикс24 и 1С) посредством исходящих REST API вызовов.
